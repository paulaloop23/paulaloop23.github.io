Accessibility Testing on City of Henderson, NV

Name: Paula Loop
Class Project: Accessibility Testing For City of Henderson, NV
Website: https://www.cityofhenderson.com/
Date: January 27, 2024

The purpose of this report is to help identify possible website digital accessibility issues in order to prevent any access issues to this website via assistive technologies. The following accessibility checklist is based on the W3C WAI Easy Checks – A First Review of Web Accessibility I hope that you will find this report helpful and informative and find the recommendations to be actionable.

Accessibility Checklist

Website Page Titles

In the web page markup, the <title> is within the <head> tag. Good page titles are particularly important for orientation, they help people know where they are and move between pages open in their browser tab. The first thing screen readers say when the user goes to a different web page is the page title.

Results: 10 pages on the City of Henderson website were evaluated for clear, unique HTML page titles. Each page has descriptive HTML title values. All titles also included with the descriptive value " and Henderson, NV" to alert users using assitive techonologies that they were still within the City of Henderson site. One suggestion would be to remove the pipes between the values to have better page title per the tips from w3.org.

Image Text Alternatives

Image text alternatives ("alt text") (pictures, illustrations, charts, etc.) Text alternatives convey the purpose of an image, including pictures, illustrations, charts, etc. The alt attribute provides an alternative text for an image, if the user for some reason cannot view it. The reason can be a slow connection, an error with the image file, or if the user uses a screen reader.

Results: There was the presense of "alt text" on most of the images but were not always as descriptive as they could be and were instead either file names or in the example of a list of icons numbered 1, 2, 3, and 4 instead of what they represent. This means that people using assistive technologies would not know what the photo or graphic is communicating. We recommend that all photos and graphics have higher quality descriptive alt text.

Text:

Headings, Contrast Ratio ("color contrast") and Resizing Text should not only support visual aesthetics but also support the navigation and visibility of text content.

Results - Headings We tested the Boards & Commissions, Police Department and the Municipal Court pages and headings tags are nested correctly. However, on those pages there were two H1 tags, there should only be one H1 per page because screen readers expect only one H1 tag.

Results - Contrast Ratio We tested the Boards & Commissions, Police Department and the Municipal Court pages and there was strong color contrast because the pages are white and the heading titles are in black. The WAVE color contrast tester did flag the main navigation as a color contrast issue because of the blue background and orange text on hover or select. Using CCA Color Contrast Analyser the main navigation failed at the AAA level at the regular text size. The CCA Color Contrast Analyser also failed the grey text on the white background as an issue at the AAA level and could be an issue for low vision users. Suggestion is to make the text a little larger or bold.

Results - Resizing Text The City of Henderson website scales up text size using keyboard commands very well. Text was increased to 200% and readability remained comprehensible on a desktop. However, there were some sections on iPhone zoom that were cutoff or created a horizonal scroll

Interaction:

Keyboard access and visual focus

Many people cannot use a mouse and rely on a keyboard to interact with web pages. Keyboard focus should be clearly visible and should follow a logical order through the page elements. Visible keyboard focus could be a border or highlight.

Results: Using only keyboard keys to navigate the City of Henderson website provided mixed results. We were able to navigate through the main navigation and search with the option to "Skip to Main Content" the area between the main navigation and the section icons on the homepage did not indicate what item we were currently on meaning that the Keyboard Focus was not obvious to us when we were using our tab and arrow keys. We recommend that the web team explore to make the City of Henderson website more keyboard-only friendly. One issue we found was on the Education page where the "Skip to Content" went directly to the form at the bottom instead of the main content at the top of the page.

Forms, labels, and errors (including Search fields):

Users are able to keyboard navigate to the different forms fields. Each form control has a label element with a for attribute that matches the value of the id attribute in the related control. For example:
  • Each form control has a label element with a for attribute that matches the value of the id attribute in the related control. For example:
    <label for="firstname">First name: </label>
    <input type="text" name="firstname" id="firstname" />
  • Each id is unique within the web page.

Results We tested the Submit Request form that was found when selecting the "ADA/508 friendly site" link from the Contact Us section because it has a form. Using only the keyboard, we were able to go to each form field and press submit. However, using the Wave tool, the Submit Request Form page has a 2 inputs on the web form that do not have labels with a related unique ID. The form itself was also flagged using Wave for having very small text.

General:

Moving, Flashing, or Blinking Content: Moving, flashing, or blinking content includes carousels (example carousel), ads, videos, auto-updating stock tickers, scrolling news feeds, and more. Users need to be able to control moving content, especially some people with attention deficit disorder or visual processing disorders.

Results: The City of Henderson has an image carousel on the homepage that auto advances very quickly and does not allow the user to take their time to read the accomanying text. There are controls to advance and reverse the carousel but not to pause. The alt text on the images are also not very descriptive and repeat the heading. We recommend that the current image carousel is replaced with an accessible image carousel. No blinking or flashing content was located on the CCC website.

Basic Structure Check: It helps you understand how some people "see" the web page differently. For this basic structure check, you look at the web page without images, styles, and layout. Web pages are often designed with multiple columns, sections, colors, and other visual aspects that help organize information for people who see the page in its default display. However, some people do not see the page this way.

Results: For this test, we used WAVE to remove all CSS styles in order to see the City of Henderson website without any visual styling. We tested the homepage and noticed that this page has a logical flow of information even without styling. Although when scrolling through the carousel area, the page kept jumping back to that section. Not sure why this is happening but this could be a problem for people navigating to items further down.
